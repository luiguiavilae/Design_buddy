**Required content areas (all must be present):**

1. **Overview** — One paragraph: what the tracking system does, what PA flow it needs, what Excel table it needs. Who this document is for.

2. **Prerequisites** — What the BCP team member needs before starting: Power Automate license, access to SharePoint/OneDrive, the Designer Buddy plugin installed, access to the plugin source code or the person who can edit it.

3. **Step 1: Create the Excel EvaluationsTable**
   - Create an Excel workbook in OneDrive/SharePoint
   - Create a Table named exactly `EvaluationsTable` (the name matters for PA)
   - Required columns in order: `fileId` (Text), `fileName` (Text), `pageName` (Text), `userName` (Text), `overallScore` (Number), `timestamp` (Text)
   - Note: `timestamp` is stored as Text (ISO 8601 string), not as an Excel date

4. **Step 2: Create the Power Automate Tracking Writer flow**
   - Trigger: When an HTTP request is received
   - Request Body JSON Schema (paste as-is):
     ```json
     {
       "type": "object",
       "properties": {
         "fileId":       { "type": "string" },
         "fileName":     { "type": "string" },
         "pageName":     { "type": "string" },
         "userName":     { "type": "string" },
         "overallScore": { "type": "integer" },
         "timestamp":    { "type": "string" }
       }
     }
     ```
   - Action: Excel Online (Business) → Add a row into a table → map each JSON field to its column
   - Response action: Status code 200, Headers `Access-Control-Allow-Origin: *` (required for CORS — without this the fetch logs a CORS error in the developer console)
   - Save and copy the HTTP POST URL from the trigger

5. **Step 3: Configure the plugin with the endpoint URL**
   - Open `src/modules/tracking/index.ts`
   - Find `export const TRACKING_ENDPOINT_URL = ''` at the top
   - Replace the empty string with the PA trigger URL
   - Rebuild the plugin (`npm run build`) and redistribute `dist/main.js` + `dist/ui.html`
   - WARNING: Never commit the real URL to the public GitHub repo. Share via Notion/Teams only.

6. **Step 4: Verify the integration**
   - Expected payload example (show the JSON shape BCP can test with Postman or Hoppscotch):
     ```json
     {
       "fileId":       "AbCdEfGhIjKl",
       "fileName":     "Onboarding BCP v3",
       "pageName":     "Pantallas principales",
       "userName":     "Ana García",
       "overallScore": 87,
       "timestamp":    "2026-03-06T14:23:11.000Z"
     }
     ```
   - Open Designer Buddy in a Figma file, run a handoff evaluation, check PA flow run history
   - Verify a new row appears in EvaluationsTable within ~30 seconds
   - If rows show `Unknown` for userName — check that the plugin binary has the `currentuser` permission in manifest.json
   - If rows show `_local` suffix for fileId — the file is a local draft not saved to Figma cloud (expected behavior)

7. **Heartbeat plan (ongoing operations)**
   - Power Automate HTTP trigger URLs expire and flows auto-disable after 90 days of inactivity
   - Every 30 days: Trigger a manual evaluation and verify a row appears in Excel
   - Assign a named owner (e.g., "Design Ops lead") responsible for the heartbeat check
   - If the flow is disabled: re-enable it from the PA portal, copy the new trigger URL if it changed

8. **URL rotation procedure**
   - If the PA trigger URL changes (flow recreated, tenant migration, etc.):
     1. Get the new URL from the PA flow trigger
     2. Update `TRACKING_ENDPOINT_URL` in `src/modules/tracking/index.ts`
     3. Run `npm run build` and redistribute the plugin binaries
     4. Verify with a test evaluation that rows appear in Excel again
   - Do NOT share the URL in any public channel or commit it to git

9. **Troubleshooting quick reference** — short table:

   | Symptom | Likely cause | Fix |
   |---------|-------------|-----|
   | No rows appearing in Excel | TRACKING_ENDPOINT_URL is empty or old | Update constant and rebuild |
   | `[Tracking]` errors in plugin dev console | CORS missing on PA Response action | Add `Access-Control-Allow-Origin: *` header |
   | All rows show `Unknown` for userName | Plugin missing `currentuser` permission | Check manifest.json |
   | All fileId values end in `_local` | Designer is working in local draft file | Normal — file not yet saved to Figma cloud |
   | PA flow run history shows no activity | PA URL domain not in manifest allowedDomains | Check `*.api.powerplatform.com` is in manifest |
